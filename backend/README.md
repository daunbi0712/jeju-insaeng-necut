# 백엔드 설정 (Google Sheets + Apps Script)

방문자/촬영 통계를 Google Sheets에 기록하는 서버리스 백엔드입니다. 별도 서버 없이
스프레드시트를 DB처럼 사용합니다.

## 1. 스프레드시트 만들기

1. [Google Sheets](https://sheets.new)에서 새 스프레드시트를 만듭니다. (이름 예: `제주 인생네컷 통계`)
2. 상단 메뉴에서 **확장 프로그램 → Apps Script**를 클릭합니다.

## 2. 스크립트 붙여넣기

1. 열린 Apps Script 편집기에서 기본 `Code.gs` 내용을 모두 지웁니다.
2. 이 저장소의 [`backend/Code.gs`](./Code.gs) 내용을 그대로 복사해 붙여넣습니다.
3. 저장합니다 (Cmd+S / Ctrl+S).

## 3. 웹 앱으로 배포

1. 우측 상단 **배포 → 새 배포**를 클릭합니다.
2. 유형 선택(톱니바퀴)에서 **웹 앱**을 선택합니다.
3. 설정:
   - 실행 계정: **나**
   - 액세스 권한이 있는 사용자: **모든 사용자** (프론트엔드에서 인증 없이 호출하기 위함)
4. **배포**를 클릭하고, 권한 요청 화면이 뜨면 본인 계정으로 승인합니다.
5. 배포 완료 후 나오는 **웹 앱 URL**(`https://script.google.com/macros/s/.../exec`)을 복사합니다.

## 4. 프론트엔드에 연결

프로젝트 루트에 `.env` 파일을 만들고 (`.env.example` 참고):

```
VITE_APPS_SCRIPT_URL=여기에_복사한_웹앱_URL
```

`npm run dev`를 다시 시작하면 촬영이 끝날 때마다 자동으로 시트에 기록되고,
화면 하단에 누적/오늘 촬영 통계가 표시됩니다.

## 시트 구조

`Log` 시트가 없으면 스크립트가 자동으로 생성합니다.

| Timestamp | Event | UserAgent |
|---|---|---|
| 2026-07-17T05:12:33.000Z | photo_taken | |

## 코드를 수정한 경우

Apps Script 편집기에서 코드를 바꾼 뒤에는 **배포 → 배포 관리 → 수정(연필 아이콘) →
새 버전 → 배포**를 눌러야 웹 앱에 반영됩니다. (저장만으로는 반영되지 않습니다.)
